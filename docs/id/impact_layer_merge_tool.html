Tool Menggabungkan Layer Dampak¶

Dalam beberapa kasus Anda mungkin ingin membuat sebuah laporan yang berisi kombinasi dari dua fungsi dampak di satu area yang sama untuk ancaman yang sama, keterpaparan yang berbeda*. Sebagai contoh Anda mungkin ingin melihat hasil kajian dari dampak banjir terhadap populasi dan bangunan dan menggabungkan keduanya kedalam satu laporan. Tool merger/menggabungkan layer dampak akan membantu anda untuk membuat hal tersebut.

Tampilan dialog Menggabungkan Layer Dampak yang telah terbuka

Prasyarat¶

Untuk menggunakan tool ini, harap diperhatikan beberapa persyaratan berikut:

  • Kedua layer dampak harus dimuat di proyek yang sedang anda buka saat ini.

  • Kedua layer dampak harus dibuat pada wilayah geografis yang sama.

  • Daerah agregrasi yang sama harus digunakan untuk kedua penilaian.

Prosedur¶

Untuk menggunakan alat ini, ikuti langkah berikut:

  • Jalankan penilaian dampak untuk suatu daerah menggunakan agregasi, misalnya bangunan terdampak banjir diagregasi berdasarkan batas kabupaten/kota.

  • Jalankan penilaian dampak kedua untuk daerah yang sama menggunakan agregasi yang sama, misalnya jumlah penduduk terdampak banjir diagregasi berdasarkan batas kabupaten/kota.

  • Buka tool Menggabungkan Layer Dampak InaSAFE dan pilih layer dampak dari daftar yang tersedia.

  • Pilih layer agregasi yang digunakan untuk menghasilkan layer terdampak pertama dan kedua.

  • Pilih direktori output.

  • Berikan tanda centang ‘Gunakan Template Laporan Terkostumisasi’ dan pilih berkas template jika anda menginginkan untuk menggunakan template anda sendiri. Harap diperhatikan bahwa semua komponen pembuatan peta yang dibutuhkan harus dipenuhi.

  • Klik OK untuk menghasilkan ringkasan kedua layer terdampak per area agregasi.

Output yang dihasilkan¶

Tool ini akan membuat PDF untuk setiap daerah agregrasi. PDF yang dibentuk akan di tempatkan pada direktori yang dituju setelah selesai proses penggabungan. Output yang dihasilkan akan terdiri dari halaman peta dan halaman tabel. Seperti yang diilustrasikan dibawah ini:

Halaman peta yang dihasilkan oleh tool penggabungkan layer dampak InaSAFE.

Halaman tabel yang dihasilkan oleh tool penggabungan layer dampak InaSAFE.

Dalam beberapa kasus dari pengkajian dampak. Saat pengguna memutuskan untuk tidak menggunakan agregrasi, hasil laporan yang dihasilkan hanya berupa satu pdf saja. Namun ketika pengguna memutuskan pengkajian dampak menggunakan agregasi, satu pdf akan dihasilkan untuk setiap daerah agregrasi.

Catatan

Setelah pembuatan laporan selesai, direktori output akan terbuka secara otomatis.

Menggunakan Template Terkostumisasi¶

Template laporan bawaan berada di /resources/qgis-composer-templates/merged-report.qpt. Jika template tersebut tidak dapat memenuhi kebutuhan anda, anda dapat menggunakan template laporan anda sendiri. Sebelum menggunakan template laporan anda sendiri, pastikan bahwa template Anda terdiri dari elemen-elemen berikut dengan ID:

  • impact-map, berjenis QgsComposerMap

  • safe-logo, berjenis QgsComposerPicture

  • summary-report, berjenis QgsComposerLabel

  • aggregation-area, berjenis QgsComposerLabel

  • map-scale, berjenis QgsComposerScaleBar

  • map-legend, berjenis QgsComposerLegend

  • organisation-logo, berjenis QgsComposerPicture

  • merged-report-table, sebuah QgsComposerHTML

Jika salah satu dari elemen tersebut tidak tersedia didalam template laporan, QGIS akan memberi Anda informasi mengenai elemen apa yang kurang dari template tersebut.

Catatan

Anda dapat mengatur elemen-elemen tersebut di segala posisi yang Anda inginkan.

Peta Template Elemen¶

Dalam ketentuan penggantian, ada tiga kelompok elemen didalam template:

1. Elements that can be changed on InaSAFE Options tool. To change the value of these elements, please go to InaSAFE Option tools and change the value of the related field. Those elements are:

  • organisation-logo. Ini sesuai dengan field Logo Organisasi pada tool pilihan InaSAFE. Jika anda tidak mengisi field ini, maka logo bawaan, supporters.png, akan digunakan.

  • Penolakan. Berpengaruh pada kolom Teks Penolakan pada tool InaSAFE Options. Jika anda tdak mengisi kolom ini, maka pengaturan bawaan akan digunakan: “InaSAFE telah dikembangkan secara bersama-sama oleh Pemerintah Indonesia-BNPB, Pemerintah Australia-AIFDR, dan World Bank-GFDRR. Instansi-instansi tersebut dan pengembang perangkat lunak dari InaSAFE tidak bertanggung jawab atas kebenaran output dari InaSAFE atau keputusan yang diberikan sebagai konsekuensinya.”

2. Elements containing tokens. The id of these element is not significant, only the token it contains. At render time, any of these tokens will be replaced. If you want to have a label containing value of these elements, enclose these elements with [] on a label i.e [impact-title] or [hazard-title]. Those elements are listed below:

  • impact-title. Menandakan judul dari dua dampak. Nilai akan menjadi “judul_dampak_pertama dan judul_dampak_kedua“

  • hazard-title. Mengindikasikan judul bencana yang digunakan untuk membuat layer dampak.

3. Elements that are direcly updated by the renderer. All of these elements below are generated automatically by the tools.

  • impact-map. Terdiri dari peta dua layer dampak.

  • summary-report. Terdiri dari ringkasan dampak dari dua layer dampak.

  • aggregation-area. Terdiri dari nama daerah agregasi.

  • map-scale. Mengindikasikan skala dari peta. Agar dapat bekerja dengan segala jenis proyeksi, kami menyarankan agar Anda menggunakan skala bar numerik.

  • map-legend. Menyajikan legenda dari penggabungan layer dampak. Pengaturan bawaan dari template legenda peta diatur kedalam 2 kolom yang menggambarkan masing-masing legenda layer dampak.

  • merged-report-table. Ini terdiri dari informasi detail dari setiap dampak.
