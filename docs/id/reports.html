Laporan¶

Laporan tentang proyek skenario dampak dibuat menggunakan QGIS template komposer. Pengaturan bawaan template sudah menjadi bagian dari plugin, tapi anda dapat dengan mudah membuat template anda sendiri di QGIS dan menggunakannya sebagai dasar dalam laporan anda.

Template lokal didukung melalui skema sederhana berikut ini:

Jika ada sebuah template tertentu yang disebut /path/to/template/foo.qpt lalu plugin terlihat di direktori template /path/to/template/ untuk sebuah file memanggil file foo-LANG.qpt, dimana “LANG” adalah kode bahasa untuk sistem lokal.

Jika sebuah file ada maka akan digunakan untuk membuat laporan sebaliknya template asli /path/to/template/foo.qpt akan digunakan

Laporan Elemen Template¶

Template ini mencakup 3 kelompok elemen selanjutnya :

  • Elemen statis.

  • Elemen yang mencakup token-token untuk penggantian.

  • Elemen yang mana ter-update secara langsung oleh para penyaji.

Elemen Statis¶

Ini adalah elemen-elemen yang tidak diubah selama pembuatan laporan. seperti beberapa logo/gambar. teks tambahan, dll. Anda dapat menghapus beberapa elemen atau menggantinya dengan yang anda inginkan jika itu diperlukan.

Elemen yang mencakup token-token untuk penggantian¶

Dalam kasus ini elemen nama tidak signifikan, hanya sebuah (beberapa) tanda yang ada. Saat ini tanda-tanda yang ada di elemen akan diganti dengan konten yang telah diterjemahkan (jika alternatif lokal berlaku) dari plugin sesuai dengan kata kunci yang tertulis pada dokumen di bawah ini.

Elemen yang mana ter-update secara langsung oleh para penyaji¶

Dalam kasus ini semua konten yang mungkin hadir di elemen seluruhnya telah diganti oleh peta realtime, walaupun pilihan gaya tampilan ini (contoh pengaturan graticule pada peta) akan menyisakan efek tertentu

Elemen ini dikenali dengan ID-nya dan plugin saat ini mendukung elemen-elemen berikut:

  • safe-logo — gambar komposer QGIS, dimana akan digunakan untuk menampilkan logo InaSAFE beserta alamat situsnya.

  • organisation-logo — gambar komposer QGIS, dimana akan digunakan untuk menampilkan logo organisasi. Kombinasi logo organisasi pendukung akan digunakan menurut pengaturan bawaan. Untuk menggunakan gambar logo anda sendiri, aturlah lokasi berkas PNG anda dialog Pilihan pada menu Plugin (lihat Pilihan untuk lebih jelasnya).

  • peta-dampak — Komposer peta QGIS, yang akan digunakan untuk menampilkan skenario dampak.

  • keterangan-dampak — Komposer Keterangan QGIS, yang akan digunakan untuk menampilkan keterangan dari skenario dampak.

  • laporan-dampak — label komposer QGIS, yang akan digunakan untuk menampilkan tabel laporan dampak.

    Catatan

    Karena komposer QGIS tidak mendukung pembesaran elemen secara otomatis berdasarkan konten, maka pastikan label telah memiliki dimensi yang cukup besar agar tabel dapat ditampilkan secara keseluruhan.

Kata kunci yang dapat diganti¶

Bagian ini mendeskripsikan tokenised keyword yang akan diturunkan pada template peta. Untuk memasukan keyword apapun pada template peta, cukup lampirkan key pada [] (misal [nama-tempat]) dan key ini akan tergantikan dengan nilai teks (misal Tondano). Daftar termasuk frasa statis yang telah di-internasionalisasikan (sehingga mereka akan ditampilkan sesuai dengan bahasa yang dipilih secara lokal pada peta, jika tidak ada terjemahan yang tersedia maka akan ditampilkan ke dalam bahasa Inggris sesuai dengan pengaturan bawaan).

Kata kunci tokenized berikut ini didukung:

  • judul-dampak — judul dari peta skenario dampak.

  • tanggal — tanggal pengukuran.

  • waktu — waktu pengukuran.

  • versi-safe — versi plugin yang digunakan.

  • disclaimer — disclaimer text. Pengaturan awal menggunakan teks: “InaSAFE telah dikembangkan bersama sama oleh Pemerintah Indonesia-BNPB, Pemerintah Australia-AIFDR dan Bank Dunia-GFDRR. Badan-badan tersebut dan individu pengembang perangkat lunak InaSAFE tidak bertanggung jawab atas kebenaran yang dihasilkan dari InaSAFEE atau keputusan yang mengikutinya sebagai konsekuensi.” Untuk menggunakan teks peringatan terkostumisasi, masukkan teks yang diinginkan pada dialog Pengaturan plugin (lihat Pilihan untuk lebih detail).

Template yang lazim digunakan¶

Anda memiliki beberapa pilihan untuk mengatur template tanpa melakukan proses pemprograman apapun. Ada 3 cara utama untuk melakukan ini:

  • Pindahkan kata kunci yang dapat diganti ke dalam elemen yang berbeda atau pindahkan seluruhnya.

  • Ubah tampilan elemen template (contoh mengatur huruf, warna, garis, dll). pindahkan elemen template itu sendiri di sekitarnya atau tambahkan/hapus seluruhnya.

  • Buat template anda sendiri dan gunakan template itu daripada template bawaan.

Template pengaturan bawaan yang disediakan seperti inasafe/safe_qgis/resources/qgis-composer-templates/inasafe.qpt dapat anda modifikasi dengan membuka template tersebut menggunakan komposer peta QGIS, membuat perubahan anda dan menimpa ulang template atau menyimpannya kembali pada direktori manapun.

Anda harus mencoba melakukan tes pada perubahan template anda sebelum menggunakannya pada lingkungan produksi.

Juga sebagai tambahan untuk template bawaan standar, anda dapat memiliki template anda sendiri sebanayak yang anda inginkan. Anda tinggal membuat template, letakan pada suatu folder di sistem berkas dan kemudian pergi ke Plugins ‣ InaSAFE ‣ InaSAFE Options atau klik pada ikon Pilihan pada toolbar plugin InaSAFE. Pada dialog pilihan, tentukan pengaturan Report template . Seharusnya berisi letak folder dengan template anda sendiri.
